# GRID Protocol Diagrams

This directory contains diagrams illustrating the core concepts, architecture, and flows of the GRID protocol.

## Diagrams

### Architecture

- **[high-level-architecture.svg](architecture/high-level-architecture.svg):** High-level overview of the GRID reference architecture.
- **[request-flow.svg](architecture/request-flow.svg):** The flow of a request through the GRID governance layer.
- **[policy-evaluation.svg](architecture/policy-evaluation.svg):** The policy evaluation process, including caching.

### Federation

- **[federation-model.svg](federation/federation-model.svg):** The GRID federation model, with multiple organizations running their own GRID nodes.
- **[cross-org-flow.svg](federation/cross-org-flow.svg):** The flow of a cross-organization request.

### Security

- **[threat-model.svg](security/threat-model.svg):** The GRID threat model.
- **[shadow-it-scenarios.svg](security/shadow-it-scenarios.svg):** Scenarios illustrating the shadow IT problem.

### Use Cases

- **[ai-agent-flow.svg](use-cases/ai-agent-flow.svg):** How GRID can be used to govern AI agent tool access.
- **[microservices-flow.svg](use-cases/microservices-flow.svg):** How GRID can be used to govern service-to-service communication.

## Usage

These diagrams can be used in documentation, presentations, and other materials to help explain the GRID protocol. They are licensed under the same license as the project (MIT License).

## Format

All diagrams are provided in SVG format, which can be easily embedded in web pages and other documents. The source files for the diagrams (e.g., Mermaid.js source) are also included in this directory.
