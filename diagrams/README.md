# GRID Protocol Diagrams

This directory contains diagrams illustrating the core concepts, architecture, and flows of the GRID protocol.

## Diagrams

### Architecture

- **[high-level-architecture.md](architecture/high-level-architecture.md):** High-level overview of the GRID reference architecture.
- **[request-flow.md](architecture/request-flow.md):** The flow of a request through the GRID governance layer.
- **[policy-evaluation.md](architecture/policy-evaluation.md):** The policy evaluation process, including caching.

### Federation

- **[federation-model.md](federation/federation-model.md):** The GRID federation model, with multiple organizations running their own GRID nodes.
- **[cross-org-flow.md](federation/cross-org-flow.md):** The flow of a cross-organization request.

### Security

- **[threat-model.md](security/threat-model.md):** The GRID threat model.
- **[shadow-it-scenarios.md](security/shadow-it-scenarios.md):** Scenarios illustrating the shadow IT problem.

### Use Cases

- **[ai-agent-flow.md](use-cases/ai-agent-flow.md):** How GRID can be used to govern AI agent tool access.
- **[microservices-flow.md](use-cases/microservices-flow.md):** How GRID can be used to govern service-to-service communication.

## Usage

These diagrams can be used in documentation, presentations, and other materials to help explain the GRID protocol. They are licensed under the same license as the project (MIT License).
